PyBridge++
==========

PyBridge++ is a Python interface to Bridge++ code set for numerical
simulation of Lattice Gauge Theory. It provides access to Bridge++
classes and functions from Python programming language.

Features
--------

  - Classes of Bridge++ are exposed to python; the class instances
    can be managed from python code, while the actual calculations
    are mostly carried out on C++ side.

  - MPI distributed-memory parallelization and multi-threading are
    supported. Interoperability with mpi4py by sharing communicator
    is available.

  - Field class objects can be exported to numpy. They are expressed
    as multi-dimensional array.


Software compatibility
----------------------

PyBridge++ has been tested with Python 3.5, 3.6, 3.8, and 3.9.

Platforms on which PyBridge++ has been tested to work include:
  - Yukawa-21 at YITP
  - Cygnus at University of Tsukuba
  - Fugaku

PyBridge++ is not known to work on the following platforms:
  - SX-Aurora at KEK
  - Flow FX1000 at Nagoya University
  - Oakforest-PACS

Quick guide
-----------

The following demonstrates the basic usage of PyBridge++, by taking 
the Wilson loop calculation as an example. It is a reduced version of
tests/wilsonloop.py.

1. Initialize

    Import pybridge module. In this example, the module is aliased
    as pyb for brevity. The module file is assumed to be located on
    python search path.
    
      >>> import pybridge as pyb

    The bridge library is initialized on importing the module.

    Next, call setup() to set lattice size and other basic parameters.

      >>> pyb.setup(lattice_size=[4,4,4,8])

    The above line specifies the size and shape of the four-dimensional
    lattice to be 4x4x4x8. The lattice_size parameter is mandatory.
    The bridge library displays the parameter values.

    The global parameters stored in CommonParameters class of Bridge++
    are accessible through properties of Common class:

      >>> nvol = pyb.Common.Nvol
      >>> ndim = pyb.Common.Ndim

    The above lines find the size of local lattice and dimensions.

2. Create objects

    A class instance is created by calling constructor as usually done
    in python. An instance of Field_G representing gauge field is thus
    generated by

      >>> u = pyb.Field_G(nvol, ndim)

    To load a gauge configuration from file, create a GaugeConfig class 
    object with file type, and invoke read_file method:

      >>> gconf = pyb.GaugeConfig("Text")
      >>> gconf.read_file(u, "conf_04040408.txt")

    WilsonLoop class implements measurement of Wilson loops. The
    parameters are passed to the constructor in key=value format,
    similar to keyword arguments.

      >>> wloop = pyb.WilsonLoop(max_spatial_loop_size=2,
      ...                        max_temporal_loop_size=4,
      ...                        number_of_loop_type=6)

3. Run measurements

    To run the measurement, invoke measure() method of the WilsonLoop
    class object with the gauge field as an argument.

      >>> r = wloop.measure(u)
      >>> print("result=", r)

4. Finalize

    The module is finalized automatically at exit of the python process,
    by calling exit() explicitly, 

      >>> exit()

    or implicitly at the end of the script.
